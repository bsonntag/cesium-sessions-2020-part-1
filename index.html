The Web

Source: MDN
Hoje vamos falar sobre JavaScript, mas antes vamos falar primeiro um pouco sobre como a Web funciona.
Um website é composto por vários ficheiros: ficheiros HTML, imagens, ficheiros de CSS e JavaScript.
Estes ficheiros têm que estar alojados num servidor web, que os serve através de HTTP.
Quando alguém quer aceder ao vosso website, terá que abrir um browser e introduzir o endereço do vosso website, ou navegar até lá a partir de um motor de pesquisa como o Google.
Ao abrir o website, o browser vai fazer vários pedidos HTTP para pedir os vários ficheiros que compoem o site. Esses ficheiros serão então enviados para o browser em resposta a esses pedidos.

Beyond the basics

Vamos então falar sobre JavaScript. Sei que já falaram sobre as bases da linguagem, por isso hoje vou falar-vos de umas coisas um pouco mais avançadas, que vocês precisam de saber para conseguirem usar React. Mas primeiro, vocês sabem porque JavaScript existe sequer?

The Web without JS

  • Static display of information
  • No interactions
  • Dynamic content must be handled by the server
Não sei se vocês sabem, mas a web começou apenas com HTML. Na altura a web era usada pela comunidade científica como uma forma de partilhar documentos, tais como teses e artigos científicos.
Usando apenas HTML, apenas se consegue fazer páginas estáticas, que servem apenas para mostrar informação.
Não é possível ter interações, como menus que abrem e fecham, nem funcionalidades como drag & drop.
Além disso, tudo o que seja conteúdo dinâmico terá que ser tratado no servidor.
Por exemplo, imaginem um site que mostra uma lista de produtos. Imaginem que essa lista é paginada, onde se mostram 10 produtos por página. Ao navegar entre páginas, todo o HTML terá que ser pedido outra vez.

The Web with JS

  • Rich interactions
  • Content can be requested by the browsers and displayed dinamically
Com JavaScript conseguimos adicionar qualquer tipo de interação aos nossos websites e aplicações web.
É também possível pedir conteúdo dinâmico ao servidor e mostrá-lo sem ter que pedir toda uma página de HTML.
Voltando ao exemplo da lista de produtos, com JavaScript é possível pedir apenas os dados da página que queremos, e substituir o conteúdo do HTML com os dados novos.

Script loading strategies

  • At end of <body>
  • <script defer>
  • <script async>
At end of <body>
  1. HTML content is parsed and displayed
  2. Script is requested
  3. Script is executed

<!DOCTYPE html>
<html>
<head>
</head>
<body>
	<h1>My website</h1>
	<script src="script.js"></script>
</body>
</html>
Using defer
  1. Script is requested and
    HTML content is parsed and displayed
  2. Script is executed

<!DOCTYPE html>
<html>
<head>
	<script src="script.js" defer></script>
</head>
<body>
	<h1>My website</h1>
</body>
</html>
Using async

The script will not block HTML parsing and will be executed as soon as possible 


<!DOCTYPE html>
<html>
<head>
	<script src="script.js" async></script>
</head>
<body>
	<h1>My website</h1>
</body>
</html>
